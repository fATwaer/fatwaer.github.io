练习1

page-translation open : cr0 和 PG bit page : 4k-byte 连续的内存单元 linear address : DIR | PAGE | OFFSET DIR是page directory的索引，PAGE是page table的索引。

寻址过程:

段部件输出一个线性地址（Linear address） Linear address(LA)

// A linear address 'la' has a three-part structure as follows:
//
// +--------10------+-------10-------+---------12----------+
// | Page Directory |   Page Table   | Offset within Page  |
// |      Index     |      Index     |                     |
// +----------------+----------------+---------------------+
//  \--- PDX(la) --/ \--- PTX(la) --/ \---- PGOFF(la) ----/
//  \---------- PGNUM(la) ----------/
//
// The PDX, PTX, PGOFF, and PGNUM macros decompose linear addresses as shown.
// To construct a linear address la from PDX(la), PTX(la), and PGOFF(la),
// use PGADDR(PDX(la), PTX(la), PGOFF(la)).

首先取这个地址的高10位作为页目录索引(Page Directory Index),大小 0x000~0xFFF,使用cr3寄存器中的高二十位定位内存中Page Directory位置。

在页目录和页表中，每个单元都是4byte。
最后寻址组成

+---------------+------------------------------+
|cr3 high 20bit | Page Directory Index * 4byte |
+---------------+------------------------------+

每个索引只使用高20位进行寻址，因为页操作的最小粒度为4KB。_4KB = 4 * 1024Byte ，1024即 4 * 2^10 ->> 0x3FF * 4 = 0xFFC_。所以只需要4个字节的前面20位进行寻址就行了，剩下的比特可以用作其他安排。 然后从线性地址中取出中间的10位作为Page Table的索引，再用之前在Page Directory中获取的高20位得到一个32位地址。得到Page Table。

Page Table高20位存储的是一个Page的4K对齐的地址，再加上线性地址中的12位(0xFFF)偏移，最后获得一个物理地址。

pte_t *
pgdir_walk(pde_t *pgdir, const void *va, int create)
{
        // Fill this function in
        size_t pgdir_index = PDX(va);
        pte_t page_table_entry = pgdir[pgdir_index];
        size_t page_offset = PTX(va);           //page table offset
        // page_table_entry is an virtual address of page table .
        if(!(page_table_entry & PTE_P)){ // this table is present ?
                // then get page table address which a pointer point to .
                if(create){
                        struct PageInfo *allocated_page = page_alloc(ALLOC_ZERO);
                        if(allocated_page == NULL) return NULL;
                        allocated_page->pp_ref++;       //reference increment
                        physaddr_t pg_phyaddr = page2pa(allocated_page) ; // get page's address
                        pgdir[pgdir_index] = pg_phyaddr | PTE_P | PTE_W | PTE_U; // permission

                } else
                        return NULL;
        }
        pte_t *pt_base = KADDR(PTE_ADDR(pgdir[pgdir_index])); // page table pointer
        return &pt_base[page_offset];
}


static void
boot_map_region(pde_t *pgdir, uintptr_t va, size_t size, physaddr_t pa, int perm)
{
        // Fill this function in
        // get the nearest number of page
        pte_t *pt_entry;
        size_t page_num = size / PGSIZE; // Size is a multiple of PGSIZE
        uint32_t i;

        for(i = 0; i < page_num; i++)
        {
                pt_entry = pgdir_walk(pgdir, (void *)va, true);
                assert(pt_entry != NULL);
                *pt_entry = pa | perm | PTE_P;
                pa += PGSIZE;
                va += PGSIZE;
        }
}


struct PageInfo *
page_lookup(pde_t *pgdir, void *va, pte_t **pte_store)
{
        // Fill this function in
        struct PageInfo *ret;
        pte_t *pt_entry = pgdir_walk(pgdir, va, false);
        if(pt_entry == NULL)
                return NULL;
        if((*pt_entry & PTE_P) == 0)
                return NULL;

        ret = pa2page(PTE_ADDR(*pt_entry));
        if(pte_store != NULL)
        {
                *pte_store = pt_entry;
        }
        return ret;
}

void
page_remove(pde_t *pgdir, void *va)
{
        // Fill this function in
        // get page table entry and data structure of the page.
        pte_t *pt_entry = NULL;
        struct PageInfo* upage = page_lookup(pgdir, va, &pt_entry);

        if(upage == NULL) return;

        page_decref(upage); //modify data structure 

        tlb_invalidate(pgdir, va);      // ?
        *pt_entry = 0 ;                 // pg table entry is set to zero.

}

int
page_insert(pde_t *pgdir, struct PageInfo *pp, void *va, int perm)
{
        // Fill this function in
        pte_t *entry = pgdir_walk(pgdir, va, true);
        if (entry == NULL) return -E_NO_MEM; // null pointer
        if (*entry & PTE_P ) {  //  this virtual address has been used 
                page_remove(pgdir, va); // decrement and free
        //      tlb_invalidate(pgdir, va);      
        }
        physaddr_t pg_physaddr = page2pa(pp);
        *entry = pg_physaddr | perm | PTE_P;
        pp->pp_ref++;

        pgdir[PDX(va)] = pgdir[PDX(va)] | perm;
        return 0;
}

内存管理总结

想了几天算是理清楚了这个内存管理机制。 首先使用i386_detect_memory函数通过CMOS设备获取内存的大小。

启动时，JOS会打印出来内存大小Physical memory: 131072K available, base = 640K, extended = 130432K，

然后表达式kern_pgdir = (pde_t *) boot_alloc(PGSIZE)在内核之后的内存块中分配一个PGSIZE(4KB)大小的内存空间，这样就先完成了Page Directory Table的建立。

kern_pgdir[PDX(UVPT)] = PADDR(kern_pgdir) | PTE_U | PTE_P这个是用来寻找页表自己使用的entry。

跟在Page Directory Table是对所有内存管理的结构体PageInfo,语句pages = (struct PageInfo *)boot_alloc(npages * sizeof(struct PageInfo))是在Page Directory Table后面分配npages * 8 bytes的内存空间。

这时候内存情况变为

| KERN | Page Directory Table | struct PageInfo |

然是是page_init()，这个函数的作用就是在结构体中标记所有已经在使用的内存页，再将所有未使用的内存页放到Struct PageInfo *pag_free_list为表头的单向链表中。

接下来是三个boot_map_region()函数，分别将UPAGE(virturl address:0xef000000)引用到之前分配的struct PageInfo, bootstarp的虚拟地址引用到CPU0's Kernel Stack, KERN~0xFFFFFFFF引用到物理地址0x00000000~0xFFFFFFF。

引用的过程中会涉及到之前填写的函数，而且在这里要注意的是虚拟地址和物理地址两个概念。 虚拟地址是指OFFSET+SELECTOR得到的地址，在JOS中，SELECTOR = 0，虚拟地址就是OFFSET，也就是指针自己本身的值，在lec5中提到virtual memory == address space / translation； 而物理地址是指CPU通过总线发送给内存进行读写的一个值。

虚拟地址是由KERN_pgdir进行管理，比如，当UPAGE作为实参进入到boot_map_region()函数中的时候，先取UPAGE的前十位作为pgdir的偏移获取一个page table entry。

如果这个entry的值是NULL，就是table不存在的情况下，就会调用boot_alloc()函数，从page_free_list链表中获取一个空内存页(4KB)来新建一个Page Table，

这个Table的实际物理地址可能在内存最后，因为pag_free_list是从前往后添加的，所以此时链表头应该是指向内存的尾部。

分配完内存页后，通过Page Translation这个机制取中间10位获得这个表的索引，得到Page entry ，然后再把调用boot_map_region()时一起传进来的(PADDR(pages))这个物理地址填到这个entry中(1bytes)。

寻址的时候本来还要加上最后的页偏移，但是UPAGE是4K对齐的，所以entry指的就是这个物理地址的页开头。pages的物理内存地址是pages这个指针减去KERNBASE的值。

因为在JOS启动的时候，entry.S就把物理地址0~4MB安装到虚拟地址的[KERNBASE, KERNBASE+4MB]，pages跟在KERN的后面。

最后KERN~0xFFFFFFFF的mapping也是因为KERN的实际.text段(code)等内容在物理地址0x10000开始处,所以在虚拟地址KERN+0X10000能正确引用物理地址0x10000的实际内容，系统内核代码在这可以正确执行。

辅助函数 str2ptr()

JOS中没有包含标准库，需要自己写一下字符串到整形的转换。

pde_t *
str2ptr(char *str)
{
    pde_t ptr;
    int temp, i; 
    size_t length;

    length = strlen(str) ;
    ptr = 0; 
    for (i = 2; str[i] != 0; i++) {

        if(str[i] <= '9') 
            temp = str[i] - '0';
        else
            temp = str[i] - 'a' + 10;
        ptr += temp * ( 1 << (length - i - 1) * 4 );
    }
    return (pde_t *)ptr;
}

        for (iter = (pde_t)pg_va; iter <= (pde_t)str2ptr(argv[2]); iter += PGSIZE) {

            if (pg_pa == (void *)0)

            else
                cprintf("0x%x: %x\n", iter, *pg_pa & ~0xFFF);
            
        }
    } else if (strcmp("set", argv[1]) == 0) {       /* set flag to Page Entry */
        pte_t flag;

        pg_va = str2ptr(argv[2]);

        if (pg_pa == (void *)0) {
            cprintf("Error: Unallocated page");

        else {
            flag = (pte_t) str2ptr(argv[3]);

            flag &= 0xFFF;          /* promise that the flag cann't affect the address */
